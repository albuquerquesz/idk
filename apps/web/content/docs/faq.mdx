## Geral

<Accordions type="single">
  <Accordion
    id="faq-1"
    title="O que é o Kubo?">
    Uma CLI opinativa que gera scaffolds de projetos TypeScript full-stack (frontend, backend, API,
    DB/ORM, auth, addons) com monorepo limpo. Veja o Início rápido na home da documentação.
  </Accordion>

<Accordion id="faq-2" title="Preciso instalar algo globalmente?">
  Não. Rode a CLI diretamente com seu gerenciador de pacotes. Veja o Início rápido e as páginas por
  comando em CLI.
</Accordion>

<Accordion id="faq-3" title="Qual gerenciador de pacotes posso usar?">
  `npm`, `pnpm` ou `bun` (todos suportados).
</Accordion>

<Accordion id="faq-4" title="Qual versão do Node.js é necessária?">
  Node.js 20+ (LTS recomendado).
</Accordion>

<Accordion id="faq-5" title="Posso usar com um projeto existente?">
  A CLI é para projetos novos. Você pode migrar aos poucos ou usar `add` para estender um projeto
  Kubo.
</Accordion>

 <Accordion 
    id="faq-6"
    title="Onde ficam os arquivos gerados?">
    Veja Estrutura do projeto para layouts de alto nível (baseado em server vs. Convex, web/native
    opcionais).
 </Accordion>
</Accordions>

## Escolhendo opções

<Accordions type="single">
 <Accordion 
    id="faq-7"
    title="A CLI recomenda uma stack?">
    Não. Escolha o que combina com suas necessidades. A CLI valida a compatibilidade. Veja CLI (por
    comando) e Compatibilidade para as regras.
 </Accordion>

 <Accordion 
    id="faq-8"
    title="Estou em dúvida entre tRPC e oRPC / Drizzle e Prisma">
    Veja Compatibilidade para orientação e restrições. Ambos os pares funcionam bem; escolha com base
    no time e nas necessidades de hospedagem.
 </Accordion>
</Accordions>

## Problemas comuns

<Accordions type="single">
 <Accordion 
    id="faq-9"
    title="Meu app mobile não conecta ao backend (Expo)">
    Defina `EXPO_PUBLIC_SERVER_URL` em `apps/native/.env` com o IP da sua máquina (não `localhost`),
    verifique o firewall ou tente `npx expo start --tunnel`.
 </Accordion>

 <Accordion 
    id="faq-10"
    title="Como desativo a telemetria?">
    Defina `BTS_TELEMETRY_DISABLED=1` (variável de ambiente do shell). Para uma execução, prefixe o
    comando; para tornar permanente, exporte no perfil do shell.
 </Accordion>
</Accordions>
