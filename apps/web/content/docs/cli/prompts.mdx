## Visão geral

A CLI usa `@clack/prompts` para perguntas interativas. Esses prompts funcionam bem na maioria dos terminais e são totalmente orientados por teclado.

## Teclas principais

- **Navegar**: setas para cima/baixo
- **Confirmar/continuar**: Enter
- **Cancelar**: Ctrl+C

## Tipos de prompt que você verá

### Seleção única (escolha um)

- **Mova** com cima/baixo, **Enter** para escolher a opção destacada.

Lugares típicos: escolher um framework web ou native, runtime ou API.

### Multi-seleção (escolha vários)

- **Mova** com cima/baixo.
- **Espaço** liga/desliga a opção destacada.
- **Enter** confirma a(s) seleção(ões).
- Alguns prompts permitem não selecionar nada (você pode pressionar Enter sem marcar nada).

Lugares típicos: selecionar tipos de projeto (web/native), escolher apps de exemplo.

### Multi-seleção agrupada (addons)

- As opções ficam organizadas sob cabeçalhos de grupo.
- **Mova** com cima/baixo, **Espaço** para ligar/desligar uma opção, **Enter** para confirmar.
- Cabeçalhos de grupo são informativos; ligue/desligue os itens dentro dos grupos.

Usado ao selecionar addons como Biome, PWA, Turborepo etc.

### Confirmar (sim/não)

- Use esquerda/direita ou cima/baixo para destacar Sim/Não e depois **Enter**.

Lugares típicos: instalar dependências, inicializar Git.

### Entrada de texto

- Digite a resposta e pressione **Enter**.
- Se a validação falhar, uma mensagem curta explica o que corrigir; edite e pressione **Enter** de novo.

Lugares típicos: nome/caminho do projeto, URLs de banco, inputs específicos de provider.

## Dicas

- Você pode pular todos os prompts com `--yes` se quiser os padrões. Veja a página de [Opções](/docs/cli/options#yes--y).
- Se começar o fluxo errado por engano, pressione **Ctrl+C** para cancelar com segurança.
