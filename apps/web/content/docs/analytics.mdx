## O que é coletado

Na criação bem-sucedida de um projeto, a CLI envia um único evento (`project_created`) com:

- Opções selecionadas (escolhas da stack), incluindo: `frontend`, `backend`, `runtime`, `database`, `orm`, `api`, `auth`, `payments`, `addons`, `examples`, `dbSetup`, `webDeploy`, `serverDeploy`, `packageManager`, `git`, `install`
- Dados de ambiente: `cli_version`, `node_version`, `platform`

Não é coletado:

- Nome do projeto, caminho ou conteúdo de arquivos (explicitamente omitidos)
- Segredos ou variáveis de ambiente da sua máquina

## Desativar telemetria

A telemetria fica habilitada por padrão. Para desativar:

```bash
BTS_TELEMETRY_DISABLED=1 npx kubojs@latest
```

<Callout title="Nota">O comando acima desativa apenas para uma execução.</Callout>

Adicione `export BTS_TELEMETRY_DISABLED=1` ao perfil do shell para tornar permanente.

## Onde ver analytics

- Gráficos: [`/analytics`](/analytics)
- Analytics compartilhados do site: [dashboard Umami](https://umami.amanv.cloud/share/pHvqHleyOl9PBfaK) (self-hosted em uma VPS Hostinger)
- Snapshot JSON bruto: `https://r2.kubojs.dev/analytics-data.json`
- Export CSV: `https://r2.kubojs.dev/export.csv`

Observações:

- Agregados são regenerados periodicamente a partir dos eventos recebidos
- Dados brutos não são expostos publicamente; a página `/analytics` apresenta apenas estatísticas resumidas

## Transparência total

Um único evento por scaffold; sem IP nem identificadores de projeto. Veja o código-fonte abaixo.

Em caso de dúvida, defina `BTS_TELEMETRY_DISABLED=1` e siga em frente. Você continua podendo usar todos os recursos da CLI.
