## Visão geral

Este projeto é um monorepo com dois apps principais:

- CLI: `apps/cli`
- Site de documentação: `apps/web`

## Setup

### Pré-requisitos

- Node.js (lts)
- Bun (recomendado)
- Git

### Instalação

```bash
git clone https://github.com/albuquerquesz/kubo.git
cd kubojs
bun install
```

## Desenvolver a CLI

```bash
cd apps/cli
# optional global link for testing anywhere
bun link
# run in watch mode (runs tsdown build in watch mode)
bun dev
```

Agora vá para qualquer outro lugar no sistema (por exemplo uma pasta de teste) e rode:

```bash
kubojs
```

Isso executa a CLI instalada localmente.

## Desenvolver a documentação

```bash
# from repo root
bun install
cd packages/backend
bun dev:setup  # you can choose local development too in prompts
```

Copie a URL do Convex de `packages/backend/.env.local` para `apps/web/.env`:

```
NEXT_PUBLIC_CONVEX_URL=http://127.0.0.1:3210/
```

Agora rode `bun dev` na raiz. Ele vai reclamar do token do GitHub, então rode isto em `packages/backend`:

```bash
npx convex env set GITHUB_ACCESS_TOKEN=xxxxx
npx convex env set GITHUB_WEBHOOK_SECRET=xxxxx
```

## Fluxo de contribuição

1. Abra uma issue/discussão antes de começar trabalho grande
2. Faça fork do repositório
3. Crie uma branch de feature
4. Faça mudanças seguindo o estilo de código existente
5. Atualize a documentação conforme necessário
6. Teste e formate

```bash
# CLI
cd apps/cli && bun dev
cd apps/cli && bun run test

# Web
bun dev

# Lint + format checks
bun check
```

7. Commit e push

```bash
git add .
git commit -m "feat(web): ..." # or fix(cli): ...
git push origin <your-branch>
```

8. Abra um Pull Request e vincule issues relacionadas

## Convenções de commit

Use mensagens de commit convencionais com o escopo apropriado:

- `feat(cli): add new CLI feature`
- `fix(cli): fix CLI bug`
- `feat(web): add new web feature`
- `fix(web): fix web bug`
- `chore(web): update dependencies`
- `docs: update documentation`
